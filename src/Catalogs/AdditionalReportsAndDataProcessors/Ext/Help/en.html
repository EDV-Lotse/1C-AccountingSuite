The Additional reports and data processors form allows you to view additional reports and data processors, as well as add, update, remove, and export them. This form is only available to users that have Full access, User, or AddChangeAdditionalReportsAndDataProcessors rights.

Each additional report or data processor is stored in a single file. These files have the following extensions:

  • .erf for additional reports
  • .epf for additional data processors
     

When you add a report or data processor to the application, you have the option to select its position in the user interface, and select the users that will have access to it.

To add a report or data processor

  1. In the Settings section, in the Parameters box, click Additional reports and data processors.
  2. Click Create, select an .erf or .epf file, and then click Open.
  3. Enter or select the following:

    • Group. The report or data processor group. You can create a new group or select from the list of available ones. This group is only used for navigation through the list of reports or data processors; it does not affect the position of the report or data processor in the application interface.
    • Description. The report or data processor name.
    • Availability. Indicates whether the report or data processor is added to the application interface. Select one of the following:
      • Enabled. The report or data processor is added to the application interface. If the report or data processor is designed to run from forms, it is visible to users that have access to these forms. If the report or data processor is designed to run from a menu, it is visible to users specified in the Users that can run the command list.
      • Disabled. The report or data processor is not added to the application interface.
      • Enabled for administrators only. The report or data processor is added to the application interface only for users that have Full access or Adding and changing additional data processors roles.
    • Responsible. The user who added the report or data processor. This field is read-only.
  4. If the report or data processor is designed to run from forms, enter or select the following:
    • Data processor assignments. The forms where the report or data processor button is available.
    • Use in list forms. Indicates whether the report or data processor button is added to list forms of affected objects.
    • Use in object forms. Indicates whether the report or data processor button is added to forms of affected objects. 
  5. If the report or data processor is designed to run from the action panel, enter or select the following:
    • Specify sections of command interface. The action panel items where the report or data processor is located. Available items are defined in the report or data processor script. Select the check boxes next to the items that you want to use.

      Selecting default list for a report adds it to the Additional reports menu item.

      Selecting default list for a data processor adds it to the Additional data processors menu item.
    • Users that can run the command. Users who have access to the command selected in the table below. Only users listed here can see the interface element that calls the command.
    • Schedule. The schedule for the command selected in the table below. It is only applicable to server method commands.
  6. Enter or select the following:
    • Commands (table). Commands available for the report or data processor. To learn about these commands, consult the report or data processor vendor.
    • Additional information. Information about the report or data processor provided by its vendor. This field is read-only.
    • Comment. Your comment to the report or data processor.
       

To update a report or data processor

  1. Obtain a new .erf or .epf file from the report or data processor vendor.
  2. Double-click the report or data processor that you want to update.
  3. Click Load File, select the new file, and then click Open.

To export a report or data processor to a file

  1. Double-click the report or data processor that you want to export.
  2. Click Unload File, specify the file name and path, and then click Save.

    This only saves the report or data processor itself; it does not save the settings specified in this form.
