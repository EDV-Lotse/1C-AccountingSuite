Items List

 

The Item List stores information about items (products or services) you buy or sell. You can create new Items as well as view, and save the list in electronic form for sending in e-mail.

Note that if you don't need to do item tracking in the system you can just type item descriptions in the Description field in sales and purchase documents without selecting Items from the Item list. There is a difference in accounting transactions in this case, you can find detailed information about the difference in the General Journal section.

 

To enter an Item:

 

1. Open the Item window by choosing Create / Item in the Lists workspace. Alternatively you can create a new Item by selecting Create in the Items list that is accessible from the Lists workspace.

2. Enter Item ID and Description.

3. Indicate if the Item is Inventory type - the system will track inventory balances, and calculate the cost based on the costing method; or Non-Inventory type - can be used for services or for items you don't want to track inventory balances or calculate cost for.

4. The system automatically prefills for you item's posting accounts when you select an item Type, you can change them if necessary.

For the Inventory item type system performs automatic reclassification of inventory cost to COGS for sales transactions.

For the Non-Inventory item type such reclassification is not performed automatically, so the system doesn't allow you to select a COGS reclass account for non-inventory items.

Detailed information about automatically generated accounting transactions in provided in the General Journal section.

5. If the South Africa financial localization is enabled select the item's purchase and sales VAT codes.

6. If the units of measure functionality is enabled indicate default purchase and sales units of measure. The units you indicate will be pre-selected when you add Items into purchasing and sales documents, and can be changed in the documents, if necessary. For the Inventory Item type both units you indicate need to have the same base unit of measure, otherwise the system won't allow you to save the Item. The base unit of measure is used for tracking in the Inventory Journal.

7. Save the Item.
