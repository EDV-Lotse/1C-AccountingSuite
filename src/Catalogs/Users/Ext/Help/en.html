The Users list stores application users with their roles and authentication methods.

To add a user

  1. In the Settings section, click Users.
  2. Click Create, and then enter or select the following:
    • Full name. The full name of the user.
    • Allow access to infobase. Indicates whether the user has the right to log on to the application.
    • Name. The user name for logging on to the application.
    • 1C:Enterprise authentication. Indicates whether the user has the right to log on to the application using their 1C:Enterprise user name and password. The user has this right is this check box is selected and the Operating system authentication check box is cleared. This is the recommended option for the administrator.
    • Password. The password for logging on to the application.
    • Confirm password. Serves for password confirmation. Enter the same password as you entered in the Password field.
    • User cannot change password. Indicates whether the user can change their password.
    • Show in user list on the logon screen. Adds the user to the list of user names on the logon screen.
    • Operating system authentication. Indicates whether the user has the right to log on to the application using their Windows domain user name and password. This is the recommended option for everyone except the administrator.
    • OS user name. The operating system user account that is used for accessing the application.
    • Interface language. The application interface language for this user.
    • Application mode. The application start mode: Auto (automatic mode selection), Ordinary application, or Managed application.
    • Roles. The list of roles that the user has.
  3. Click Write and close.
