Cash Sales

 

The Cash Sale document incorporates the goods issue, invoice, and receipt sales steps into one transaction. It's essentially an ad hoc sales document. You can create new Cash Sales as well as view, print, and save them in electronic form for sending in e-mail.

 

To enter a Cash Sale:

 

1. Open the Cash Sale window by choosing Create / Cash Sale in the Sales workspace. Alternatively you can create a new Cash Sale by selecting Create in the Cash Sales list or in the Sales Documents journal (both accessible from the Sales workspace).

2. Change the default date, if necessary. The date is used for General Journal and Inventory Journal posting.

3. Indicate whether the payment is deposited to a bank account or deposited later individually or with other payments using the Deposit document.

Select the account that will receive the funds (the field is available for editing only if the Account option is selected in the Deposit Type box).

Select the Payment Method.

4. Select a Customer. For sales transactions in the Customer field you can only select companies of the Customer type.

If the multi-currency functionality is enabled after you select a customer its default currency will appear in the currency field on a Cash Sale and the system will calculate an exchange rate. The default currency and the calculated exchange rate can be changed, if necessary. Also the system recalculates the exchange rate if you change the Cash Sale date. If you change the currency the system won't recalculate already entered prices in the line items, so it's advised to change the currency and the Cash Sale date before filling in the line items.

5. If the warehouse locations functionality is enabled select a warehouse in the Location field.

6. Fill in the line items of a Cash Sale by selecting Items.

If the units of measure functionality is enabled when you select an Item its default sales unit of measure (U/M) will appear in the U/M field. After entering the Quantity U/M, the system will recalculate the Base Quantity of the Item. Note, that all inventory tracking in the Inventory Journal is done in the base unit of measure.

The system will automatically pick a price from the Price List when an Item is selected. An Item price in the Price List is presumed to be set on the default sales unit of measure of the Item in the default currency. If you change the default unit of measure for the line item the system will recalculate the quantity but you'll need to enter a price manually.

7. For the US financial localization the system automatically calculates sales tax for each Item line, and the total amount for the Cash Sale. The sales tax is calculated based on the buyer's sales tax code selected in the Company window. The system automatically selects "taxable" for inventory item types, and "non-taxable" for non-inventory item types (services), which can be changed for each line item if necessary. When the Cash Sale is posted the system also records a sales tax liability in the General Journal.

If the South Africa financial localization functionality is enabled the system will automatically calculate VAT based on VAT codes assigned to items. You can change individual VAT codes on items, if necessary. When the Cash Sale is posted the system also record a VAT liability in the General Journal.

8. Save the Cash Sale. The system will post it in both the General Journal and the Inventory Journal. Detailed information about generated posting transactions is provided in the General Journal section.
