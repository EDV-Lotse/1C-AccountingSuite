Sales Invoices

 

A sales invoice created within the sales transaction workflow (based on a Goods Issue document created from a Sales Order) won't create any General Journal or Inventory Journal postings, but will register the fact of invoicing the customer. You can see the invoiced quantity in the Invoiced column of the respective Sales Order.

A sales invoice created outside the sales transaction workflow (not based on a Goods Issue document) records an asset (customer's liability to you), income from the transaction, calculates cost of goods sold (COGS), and updates inventory balances.

Note that the system won't allow you to create a Sales Invoice based on a Sales Order if there is a Goods Issue created based on the same Sales Order. In this case create a Sales Invoice based on the Goods Issue.

You can create new Sales Invoices as well as view, print, and save them in electronic form for sending in e-mail.

 

The following documents can be created based on a Sales Invoice:

·         Sales Return

 

A Sales Invoice can be created based on:

·         Sales Order

·         Goods Issue

 

 

To enter a Sales Invoice:

 

1. Open the Sales Invoice window by choosing Create / Sales Invoice in the Sales workspace. Alternatively you can create a new Sales Invoice by selecting Create in the Sales Invoices list or in the Sales Documents journal (both accessible from the Sales workspace).

2. Change the default date, if necessary. The date is used for General Journal and Inventory Journal posting.

3. Change invoice Terms, the system will automatically recalculate the Due Date, that can be changed, if necessary.

4. Select a Customer. For sales transactions in the Customer field you can only select companies of the Customer type.

If the multi-currency functionality is enabled after you select a customer its default currency will appear in the currency field on a Sales Invoice and the system will calculate an exchange rate. The default currency and the calculated exchange rate can be changed, if necessary. Also the system recalculates the exchange rate if you change the Sales Invoice date. If you change the currency the system won't recalculate already entered prices in line items, so it's advised to change the currency and the Sales Invoice date before entering line items.

5. If the warehouse locations functionality is enabled select a warehouse in the Location field.

6. Enter line items of a Sales Invoice by selecting Items.

If the units of measure functionality is enabled when you select an Item its default sales unit of measure (U/M) will appear in the U/M field. After entering the Quantity U/M, the system will recalculate the Base Quantity of the Item. Note, that all inventory tracking in the Inventory Journal is done in the base unit of measure.

The system will automatically pick a price from the Price List when an Item is selected. An Item price in the Price List is presumed to be set on the default sales unit of measure of the Item in the default currency. If you change the default unit of measure for the line item the system will recalculate the quantity but you'll need to enter a price manually.

7. For the US financial localization the system automatically calculates sales tax for each Item line, and the total amount for the Sales Invoice. The sales tax is calculated based on the buyer's sales tax code selected in the Company window. The system automatically selects "taxable" for inventory item types, and "non-taxable" for non-inventory item types (services), which can be changed for each line item if necessary. When the Sales Invoice is posted the system also records a sales tax liability in the General Journal.

If the South Africa financial localization functionality is enabled the system will automatically calculate VAT based on VAT codes assigned to items. You can change individual VAT codes on items, if necessary. When the Sales Invoice is posted the system also record a VAT liability in the General Journal.

If the Brazil financial localization functionality is enabled the system calculates the ICMS tax (Imposto sobre Operacoes Relativas a Circulacao de Mercadorias e sobre Servicos) for each line item based on the Item's ICMS group, entered CFOP (codigo fiscal de operacoes e prestacoes) transaction classification, and the ICMS tax table.

8. Save the Sales Invoice. If the Sales Invoice is created based on a Goods Issue document the system won't generate any General Journal or Inventory Journal postings. If the Sales Invoice isn't based on a Goods Issue document the system will post it in both the General Journal and the Inventory Journal. Detailed information about generated posting transactions is provided in the General Journal section.
