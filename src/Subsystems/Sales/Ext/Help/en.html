Overview

 

The Sales workspace in conjunction with the other system workspaces supports automating sales business processes - quoting, tracking sales orders, goods issue, invoicing, accepting payments, and customer returns. Automatic document posting updates inventory balances and creates general journal transactions. You can generate individual document print forms, PDF/Excel versions of documents for sending via e-mail, as well as sales reports, such as A/R Aging.

The full implemented sales transactions workflow is Sales Quote -> Sales Order -> Goods Issue -> Sales Invoice -> Receipt.

The system also includes a standalone document - Cash Sale that incorporates the goods issue, invoice, and receipt sales steps into one transaction. The Sales Return document is used to track returns.

By default the Sales Quote, Sales Order, Sales Invoice, Cash Sales, and Sales Return documents are located in the Sales workspace, whereas the Goods Issue documents in the Warehouse workspace, and the Receipt document in the Banking workspace. This default configuration can be changed to accommodate user requirements.

The system supports both real-time posting (updating inventory and creating general journal transactions), as well as batch posting. The following sales documents generate posting - Goods Issue, Sales Invoice, Receipt, Cash Sales, and Sales Return. The Weighted Average, FIFO and LIFO costing methods are used for inventory costing. Detailed information about accounting transactions is provided in the General Journal section.
